# argolizm — サービス仕様書

## 1. サービス概要
- **一言で**: エンジニア・デザイナー等クリエイティブ職の市場価値を自動算定するオープンアルゴリズム
- **対象ユーザー**: フリーランスエンジニア・デザイナー、エージェント担当者、TalentFlow利用企業
- **解決する課題**: 職歴書・GitHubアクティビティ・ポートフォリオを総合的に評価し、主観に頼らない市場価値の定量化
- **ステータス**: MVP（プロトタイプ）

## 2. ユーザーロールと権限
| ロール | できること |
|---|---|
| 誰でも | PDF職歴書アップロード、GitHub連携、ポートフォリオURL入力、スコア算出 |

認証不要。全機能がパブリックに利用可能。

## 5. データモデル
サーバーサイドDB不使用。全データはクライアントのみで処理。

### スコア型（`src/lib/scorer.ts`）
- `TalentScore` — total, technical, achievement, growth, fit(null)
- `GitHubProfile` — GitHub REST API レスポンスのサブセット
- `ScorerInputs` — resumeText, github, portfolioUrls

## 6. 外部連携
- PDF解析: `pdfjs-dist` v4（クライアントサイド）
- GitHub: `https://api.github.com/users/{username}` (公開API、認証不要)
- ホスティング: 未定（Vercel 想定）

## 7. ビジネスルール
- スコアはすべてフロントエンドで計算。外部サーバーへ個人データは送信しない
- モックスコアである旨を画面上に明示
- GitHub API はレート制限あり（未認証: 60 req/hour）

## 8. 非機能要件
- 想定ユーザー数: 社内利用・デモ用（<100 users）
- セキュリティ: 個人データ非送信、PDFはブラウザ内のみで処理

## 9. 既知の課題・制限
- pdfjs-dist の Worker を CDN から読み込むためオフライン不可
- スキャンPDF（画像のみ）はテキスト抽出不可
- モックスコアの計算ロジックは簡易的。実用精度には達していない
